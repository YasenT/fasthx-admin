Registered Tools

What are tools?

Tools are Python functions the AI can call to look up live data from your app — database queries, external API calls, or computations. For example, a find_customer tool lets the AI answer "What's John Smith's account number?" by running a real query instead of guessing.

Enable only the tools you want the AI to have access to. Tools are defined in application code with @tool_registry.tool() and cannot be added from this page.

Lifecycle Hooks

What are hooks?

Hooks are Python functions that run at specific points in the chat flow. Four events are supported:

  • session_start — when a new chat session begins (good for audit logs)
  • user_prompt_submit — before the user's message is sent to the AI (inject context, redact PII)
  • pre_tool_use — before a tool runs (gate by user role, rate-limit; return a message to block the call and feed it back to the AI)
  • post_tool_use — after a tool runs (transform or redact the result)

Hooks are defined in application code with @tool_registry.on("event") and cannot be added from this page. A failing hook never blocks the chat — exceptions are logged and the hook is skipped.
